# Advanced Retrieval

超越朴素向量检索的增强型检索方法，关注多阶段召回、跨步推理、结构化导航与结果重排。

## 定位

这里承接 `02-retrieval/` 下较复杂的检索机制：它们不再只是单点组件，而是开始涉及多阶段流程、跨块关联、路径选择或结构感知。

本目录按“检索如何变得更强”来组织，不按论文、项目或工具类型组织。

- 如果你想找**混合不同召回信号**的方法，进入 `hybrid-retrieval/`（同时承接检索策略组合与搜索方式组合）
- 如果你想找**跨多步证据链**的检索，进入 `multi-hop-retrieval/`
- 如果你想找**递归展开上下文**的检索，进入 `recursive-retrieval/`
- 如果你想找**对候选结果再排序**的方法，进入 `reranking/`
- 如果你想找**面向长文档结构导航**的代表性对象研究，进入 `page-index/`

## 边界说明

- 这里讨论的是增强型检索机制，不承接分块、嵌入、向量库这类基础组件；这些内容留在 `../chunking-strategies/`、`../embedding-models/`、`../vector-databases/`
- 这里也不承接 RAG 的整体范式综述、Agent 驱动流程或知识编译范式；这些内容留在 `../../03-advanced-patterns/`
- `page-index/` 是对象研究目录，不是与同层目录完全对称的通用主题节点；它放在这里，是因为它高信号承载了结构化导航检索这一重要思路

## 子目录

| 目录 | 回答的问题 | 当前状态 |
|------|------------|----------|
| `hybrid-retrieval/` | 如何组合多种检索信号、检索器或召回阶段，并把关键词、稀疏信号、稠密向量等搜索方式纳入同一检索流程？ | 建设中 |
| `multi-hop-retrieval/` | 当答案依赖跨段落、跨文档或跨实体链路时，检索如何逐步推进？ | 目录已建立 |
| `recursive-retrieval/` | 当一次检索不足以覆盖上下文时，如何按层级或局部结果继续展开？ | 目录已建立 |
| `reranking/` | 初步召回之后，如何重新排序以提升最终上下文质量？ | 目录已建立 |
| `page-index/` | 长文档结构树索引与推理导航检索框架的对象研究入口 | 有正文 |
