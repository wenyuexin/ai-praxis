# candidates.md：LLM Wiki 候选研究对象

本目录记录下一步值得继续研究的候选对象。这些对象来自一轮去锚点调研（`temp/web-search/5.md` / `6.md` / `7.md` / `8.md`），只收录与 llm-wiki 当前研究边界相关、且可作为后续对象研究或比较参照的项目。条目分为**核心候选样本**（与 llm-wiki 当前边界高度相关）和**边界参照对象**（有助于澄清边界，但不作为同等中心的候选）。

---

## 核心候选样本

### GraphRAG（Microsoft, 2024）

- **类型**：结构化知识组织系统
- **关联问题**：查询前知识组织、图社区摘要、全局查询
- **研究理由**：与 llm-wiki 共享"查询前组织知识"这一理念，但采用图拓扑而非文档编译路径。适合作为同一主问题的不同技术路线的对照样本。
- **来源**：6.md / 8.md

### STORM（Stanford）

- **类型**：主动知识合成引擎
- **关联问题**：查询前知识编译、多视角合成、知识资产生成
- **研究理由**：在用户查询前主动将互联网/企业数据编译为结构化知识资产，与 llm-wiki 的"摄入时编译"在时间维度和目标上高度重叠，但产物形态和方法不同。
- **来源**：8.md

### Dify Knowledge 模块

- **类型**：工程实践（知识管理单元）
- **关联问题**：知识编译层、知识产物的一等公民化、多格式导入
- **研究理由**：把"知识库"从 RAG 管道中抽离为独立的知识管理单元。体现了"knowledge as a first-class artifact"的工程落地，与 llm-wiki 理念一致但形态不同。
- **来源**：6.md

### KBLaM

- **类型**：非检索知识注入
- **关联问题**：非检索型外部知识传递、key-value attention 注入
- **研究理由**：代表了"检索不是外部知识传递的唯一方式"这一方向。与 llm-wiki 的"编译即用"理念有部分重叠但路径不同，适合作为替代路径的对照样本。
- **来源**：5.md

### Sleep-Time Compute Paradigm

- **类型**：理论框架（离线预计算）
- **关联问题**：离线知识预组织、查询时计算节省、知识组织的时机
- **研究理由**：系统化了"预先组织知识"的理论框架，与 llm-wiki 的"摄入时编译"共享知识组织的时机前置这一核心理念。可作为理论基线的参考。
- **来源**：5.md

## 边界参照对象

### Memoria（Agentic Memory Framework）

- **类型**：持久化记忆框架
- **关联问题**：跨会话知识持久化、Agent 记忆的可解释性、知识图用户建模
- **研究理由**：将持久化记忆作为 Agent 的一等能力，与 llm-wiki 共享"知识不应在会话间归零"的认知。帮助定位 llm-wiki 与持久化记忆框架的技术重合点与差异。
- **来源**：5.md

### Collaborative Memory

- **类型**：多用户/多 Agent 记忆共享框架
- **关联问题**：多 Agent 知识共享、权限治理、跨用户记忆分层
- **研究理由**：首次将知识基础设施引入多用户权限与治理维度。当前 llm-wiki 完全缺失此维度，了解该项目有助于判断"共享知识与治理"是否属于 llm-wiki 的合理扩展方向。
- **来源**：5.md
