# Interdisciplinarity Roadmap

> 如果你不确定从哪开始，或想知道不同身份的读者应该如何阅读 interdisciplinarity/ 目录，从这里开始。
> 边界：本文件回答"从哪开始、按什么路径推进"，不替代 `README.md` 的目录定位说明，也不替代 `index.md` 的结构展示与查找导航。

## 按读者背景推荐起点

| 读者背景 | 建议起点 | 理由 |
|---|---|---|
| ML 工程师 / 系统架构师 | [04 控制论与系统论](./04-cybernetics-systems/) 或 [06 社会学与组织管理](./06-sociology-and-organization/) | 反馈回路、组织协调等概念与 Agent 架构直接对应 |
| 研究人员 / 理论探索者 | [03 心灵哲学](./03-philosophy-of-mind/) 或 [02 语言学与语用学](./02-linguistics-and-pragmatics/) | 深入探讨能力边界、语义基础和理解的本质 |
| 产品经理 / 策略制定者 | [07 经济学与博弈论](./07-economics-game-theory/) 或 [08 法学与治理](./08-law-and-governance/) | 激励设计、合规框架与治理机制 |
| 通用读者 | [01 认知科学与神经科学](./01-cognitive-neuroscience/) | 概念最贴近直觉，表格化程度最高 |

## 按阅读目标推荐路径

- **想在具体技术领域中找到跨学科视角的对应关系**：从 `index.md` 的"按问题找入口"表格开始，直接定位到目标子目录和关联目录
- **想系统了解一个跨学科方向的全貌**：先进入对应子目录；如该目录已有 `README.md` 或 `overview.md`，优先从那里开始，再深入具体专题
- **想对比多个学科视角对同一 AI/ML 问题的看法**：先确定问题归属的技术目录（如 Multi-Agent），再从 index.md 中查找被哪些跨学科条目引用
- **想快速理解映射方法论**：先读以下"映射方法论"部分，再选择感兴趣的学科子目录

## 映射方法论

在阅读或撰写任何跨学科笔记时，可围绕以下框架建立映射：

1. **这个学科的核心张力是什么？**（如：效率 vs 柔性、个体理性 vs 集体最优）
2. **该张力在 LLM / ML 系统中有无同构表现？**（如：推理延迟 vs 深度、探索 vs 利用）
3. **该学科发展出了哪些解决方案或理论模型？**（如：组织双元性、机制设计）
4. **这些解决方案能否转译为工程架构或训练策略？**（如：动态路由、奖励塑形）

> 撰写新笔记时，建议遵循 [`template.md`](./template.md) 的结构化模板。
