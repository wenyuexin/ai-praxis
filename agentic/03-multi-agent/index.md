# 03 Multi-Agent Index

> 如果你已经知道自己要找的是多智能体中的哪类协作、通信、协调或共享状态问题，从这里开始。
> 边界：本文件负责目录结构展示与查找导航，不替代 `README.md` 的目录定位说明，也不替代 `overview.md` 的整体理解主线。

## 目录结构

```text
03-multi-agent/
├── collaboration/              # 协作模式与角色分工
├── coordination/               # 协调机制与执行协同
├── communication-protocols/    # 通信协议与消息结构
├── shared-state-and-context/   # 共享状态与上下文管理
├── organizational/             # 组织结构与多智能体架构
├── competition-and-conflict/   # 竞争、冲突与协调失败
├── overview.md                 # 多智能体整体理解主线
├── backlog.md                  # 当前内容缺口
└── conflict.md                 # 已记录的口径冲突
```

## 按问题找入口

- 想看多 Agent 如何分工、协作或共同完成任务：先看 `collaboration/`
- 想看任务协调、调度、同步或局部/全局协同：先看 `coordination/`
- 想看 Agent 之间如何通信、传递消息或约定协议：先看 `communication-protocols/`
- 想看共享上下文、共享状态、状态污染或上下文同步：先看 `shared-state-and-context/`
- 想看组织结构、角色层级、团队式 Agent 架构：先看 `organizational/`
- 想看竞争关系、冲突处理或多 Agent 失效模式：先看 `competition-and-conflict/`

## 按阅读目标找入口

- 想先恢复多智能体的整体理解主线：读 [`overview.md`](./overview.md)
- 想看当前还有哪些内容缺口：读 [`backlog.md`](./backlog.md)
- 想看当前已记录的冲突：读 [`conflict.md`](./conflict.md)
- 想回到 Agentic 目录整体导航：读 [`../index.md`](../index.md)
