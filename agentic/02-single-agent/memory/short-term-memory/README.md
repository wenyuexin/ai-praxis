# Short-Term Memory

> 适用范围：单智能体短期记忆与上下文保持研究
> 阶段状态：调研优先，暂不收敛实现方案

## 一句话定位

本目录用于整理单智能体在单次任务或短时间跨度内维持上下文、工作状态与临时信息的机制。

## 快速入口

- 结合上层 `../README.md` 理解 short-term memory 在整体 memory 体系中的位置。
- 重点关注上下文窗口利用、工作记忆组织、临时状态更新与丢失风险。
- 如出现术语定义、机制边界或版本口径冲突，记录到 `../../../temp/conflict.md`。

## 边界说明

- 放在这里：工作记忆、短时上下文保持、临时状态、任务内信息组织。
- 不放在这里：跨任务长期存储（放 `../long-term-memory/`）、检索算法与召回策略（放 `../retrieval-methods/`）。

## 同目录导航

- 建议结合 `../long-term-memory/README.md` 对比短时上下文保持与长期持久化存储。
- 建议结合 `../retrieval-methods/README.md` 区分当前工作状态维护与外部记忆召回。
