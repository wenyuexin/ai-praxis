# Retrieval Methods

> 适用范围：单智能体记忆检索与召回机制研究
> 阶段状态：调研优先，暂不收敛实现方案

## 一句话定位

本目录用于整理单智能体如何从已有记忆中检索、召回并选择当前任务真正需要的信息。

## 快速入口

- 结合上层 `../README.md` 理解 retrieval methods 在 memory 体系中的角色。
- 重点关注召回触发条件、检索策略、排序机制、上下文注入与过时信息过滤。
- 如出现检索定义、评价口径或版本差异冲突，记录到 `../../../temp/conflict.md`。

## 边界说明

- 放在这里：检索触发、召回策略、排序与过滤、上下文注入、检索评估。
- 不放在这里：记忆内容本身的长期存储（放 `../long-term-memory/`）、短期工作记忆维护（放 `../short-term-memory/`）。

## 同目录导航

- 建议结合 `../long-term-memory/README.md` 查看长期记忆内容如何进入检索流程。
- 建议结合 `../short-term-memory/README.md` 对比外部召回与上下文内工作记忆的不同职责。
