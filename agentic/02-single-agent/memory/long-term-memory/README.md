# Long-Term Memory

> 适用范围：单智能体长期记忆与跨任务经验积累研究
> 阶段状态：调研优先，暂不收敛实现方案

## 一句话定位

本目录用于整理单智能体如何跨任务保存经验、沉淀偏好、维护长期知识，并在后续任务中进行有效调用。

## 快速入口

- 结合上层 `../README.md` 理解 long-term memory 与 short-term memory、retrieval-methods 的关系。
- 重点关注跨任务持久化、经验写回、用户偏好、知识衰减与污染控制。
- 如出现记忆定义、存储边界或版本口径冲突，优先记录到上层 `../conflict.md`；若只是临时判断，可先做临时冲突整理，但不要长期替代稳定冲突入口。

## 边界说明

- 放在这里：长期持久化记忆、跨任务经验、偏好保存、记忆更新与遗忘机制。
- 不放在这里：任务内临时状态（放 `../short-term-memory/`）、检索方法本身（放 `../retrieval-methods/`）。

## 同目录导航

- 建议结合 `../short-term-memory/README.md` 对比跨任务持久经验与任务内工作状态。
- 建议结合 `../retrieval-methods/README.md` 查看长期记忆如何被检索与注入当前任务。
