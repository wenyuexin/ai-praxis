# Codex Agent（代码能力）

围绕 **Codex 作为代码 Agent 本体** 的系列文档，聚焦主线理解、机制展开、使用方法与证据核验。

---

## 该从哪开始读？

### 第一次接触 Codex

如果你还不确定 Codex 是什么、值不值得花时间了解：

1. **先读** `overview.md` —— 先获得对 `Codex` 产品形态、能力边界、运行环境和开源边界的整体认识
2. **再读** `codex-agent-overview.md` —— 把焦点收缩到“Codex 作为代码 Agent 是什么”
3. **继续读** `codex-agent-mechanisms.md` —— 理解执行闭环、治理机制与环境边界
4. **按需延伸** 到 `codex-agent-practice.md`、`codex-agent-evidence.md` 或 `codex-official-docs.md`

### 打算用 Codex 做开发

1. `codex-agent-overview.md` —— 先确认它适合承担什么类型的开发协作
2. `codex-agent-mechanisms.md` —— 理解原理、闭环、工具和权限机制
3. `codex-agent-practice.md` —— 掌握适用场景、任务范式和常见反模式
4. `codex-troubleshooting.md` —— 遇到问题时快速排查

### 做选型评估或深度研究

- `codex-agent-evidence.md` —— 关键事实、证据分级、时间轴
- `codex-cloud-sandbox.md` —— Cloud 执行与本地沙箱的区别
- `codex-computer-use.md` —— GUI 远程操控能力
- `codex-automations-ci.md` —— Automations 和 CI/CD 集成
- `codex-open-source.md` —— 开源状态与本地部署边界

### 技术爱好者 / 源码研究者

- `codex-architecture.md` —— Rust 源码架构
- `codex-multi-agent.md` —— Subagent、线程生成、并行协作与工作域边界
