# Codex Computer Use（远程操控 GUI）

> **专题文档**。本文介绍 Computer Use（远程操控 GUI）的能力边界、使用案例和安全模型。适合评估 GUI 自动化能力时查阅。不属于主线阅读路径。



> 基于 `developers.openai.com/codex/app/computer-use` 官方文档 + 社区实测整理（2026.5 最新）。

---

## 1. 官方能力边界

| 维度 | 能做 ✅ | 不能做 ❌ | 证据等级 |
|------|--------|----------|---------|
| **浏览器操控** | 点击元素、输入文本、滚动页面、切换 Tab、截图 | 无法处理 CAPTCHA、无法下载文件到本地磁盘 | A-官方 |
| **表单操作** | 填写表单、提交表单、多步骤表单流程 | 无法处理需要短信/邮件验证的表单 | A-官方 |
| **跨页面导航** | 点击链接跳转、前进/后退、URL 直接导航 | 无法处理需要持久登录态的 SSO 跳转（session 会断） | B-社区 |
| **页面批注** | 在页面上标注、高亮、添加注释 | 无法直接编辑页面 DOM（只能模拟用户操作） | A-官方 |
| **跨服务操作** | 在浏览器内完成多服务间跳转（如 GitHub → Jira → Slack） | 无法调用原生 API（只能通过 UI 操作） | B-社区 |
| **文件系统** | 可以读取页面内容、截图保存 | ❌ **不能**直接读写本地文件系统（这是和 Local 模式的核心区别） | A-官方 |

> ⚠️ **关键认知**：Computer Use 不是"远程桌面"，而是"AI 操控浏览器的手"。它的边界是 **浏览器视口内的一切用户能做的事**，但不包括浏览器外的文件系统和需要持久认证的服务。

## 2. 实际使用案例

| 场景 | 案例描述 | 效果 | 证据等级 |
|------|----------|------|---------|
| **批注页面** | 在 PR 页面逐行添加评论，指出具体代码问题 | 精准定位到行号，评论格式规范 | B-社区 |
| **填写表单** | 自动填写 Jira Ticket 表单（摘要/优先级/指派人） | 节省 5-10 分钟/单 | B-社区 |
| **跨服务操作** | GitHub PR → Jira 创建关联 ticket → Slack 通知频道 | 全流程自动化，无需人工跳转 | B-什么值得买 |
| **多网页数据爬取** | 爬取多个网页 → 按规则分析 → 生成研究报告 | 数小时流程稳定完成 | C-CSDN |
| **远程操控 GUI** | 2026.4.16 上线，可批注页面、点击元素、填写表单 | 官方主推场景 | A-官方 |

## 3. 安全模型

| 维度 | 实现方式 | 证据等级 |
|------|----------|---------|
| **认证** | SSH Key 认证 + API Key 双重认证 | A-官方 |
| **权限控制** | `approval_policy` 四级控制，Computer Use 操作默认需要确认 | A-官方 |
| **网络隔离** | Computer Use 运行在独立浏览器实例，网络流量受控 | A-官方 |
| **操作审计** | 每个操作（点击/输入/截图）都有日志记录 | A-官方 |
| **安全评估** | ⭐ **有条件可用** — 双重认证 + 审批策略，但浏览器自动化本身有被滥用风险 | A |
