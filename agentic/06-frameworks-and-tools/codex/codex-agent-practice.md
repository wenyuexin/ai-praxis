# Codex Agent 使用方法

> 目标：提供可复用的使用方法，确保“可持续提效”，而不是一次性技巧。

## 1. 方法主线

推荐固定流程：

1. 定义任务目标（要产出什么）
2. 限定上下文边界（能用哪些信息）
3. 明确约束（性能/兼容/安全）
4. 规定输出格式（计划、diff、测试）
5. 设置验收标准（通过条件）

## 2. 三类高频任务范式

### A. 需求到实现

- 先让 Agent 给方案对比
- 再选一个最小可行实现
- 最后绑定验收与回归测试

### B. 代码改造

- 先生成变更计划
- 再按模块分批改动
- 每批改动都跑验证

### C. 问题定位

- 输入最小复现信息
- 要求根因排序与验证步骤
- 修复建议必须带测试

## 3. 反模式（避免）

- “一次性重构整个仓库”
- “不给约束直接要最优”
- “聊天结果直接提交”
- “把长上下文当长期记忆”

## 4. 稳定性与成本控制

- 把长任务拆成“分析→实现→验证”
- 对输出长度和轮次设置上限
- 记录每轮关键决策与结果
- 对重复任务沉淀固定模板

## 5. 团队落地最小链路

- PR 前：Agent 预审风险
- CI 中：测试 + SAST 强制门禁
- PR 中：人工 review 语义与边界
- PR 后：规则沉淀到团队指令
