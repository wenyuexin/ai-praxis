# 反思层（`reflections.md`）的设计意图

本文件不重复 `../research-artifacts.md` 的条文，而是解释：**为什么研究目录中应允许存在一层不以事实闭环为第一目标、但又不应混进正文和 Evidence 的“反思层”。**

## 1. 这个文件回答什么问题

`research-artifacts.md` 负责回答正文、机制专题和研究辅助材料怎么分层。

本文件回答的是更细的一层：

- 为什么研究目录不应只有“正文 / 证据 / 源码笔记”三种内容
- 为什么基于客观现状的自由发散也需要稳定容器
- 为什么这类内容既不能直接进正文，也不该被压成纯 Evidence 或 source notes
- 为什么它应作为研究辅助材料层的一种，而不是新建一套平行元信息系统

## 2. 为什么研究需要一层允许自由发散的空间

科研和深入研究并不只靠已证实结论推进。

很多真正有价值的推进，首先出现为：

- 对某个对象的直觉判断
- 对设计选择的怀疑
- 对结构边界的阶段性理解
- 尚未证实但值得继续追踪的假说
- 一种“这里有问题 / 这里有机会”的感受

如果没有稳定容器，这些内容通常会出现两种坏结果：

- 要么被直接写进正文，过早冒充稳定知识
- 要么因为“不够正式”而无处安放，最后丢失

因此，需要一层既允许保留这些内容、又不让它们污染正文的反思层。

## 3. 为什么不能把这些内容塞进 `source.md` 或 `evidence.md`

`source.md` 和 `evidence.md` 都太窄了。

- `source.md` 更偏“我看到了什么源码 / 材料事实”
- `evidence.md` 更偏“这个判断凭什么成立”

而反思层承接的是另一类东西：

- 我当前怎么理解这个对象
- 我为什么觉得这个设计选择可能更关键
- 哪个问题值得继续追
- 哪种解释目前最有启发，但还不能当结论写

如果把这些内容硬塞进 `source.md` 或 `evidence.md`，会让辅助材料层重新失去分工。

## 4. 为什么它应放在 `notes/` 里，而不是额外升格

反思层需要稳定容器，但不需要独立升格成新的正文层或元信息层。

原因是：

- 它仍然依附具体研究目录而存在
- 它服务的是后续研究推进，不是普通读者的完成态阅读
- 它天然允许不闭环、带主观视角、保持阶段性

所以它最合适的落位是：

- 已有 `notes/` 时，放在 `notes/reflections.md`
- 辅助材料还很少时，可临时在目录根下用 `reflections.md`

## 5. 为什么这层可以更自由，但仍要有最低约束

反思层的价值就在于比 Evidence 和正文更自由。

但如果完全没有约束，它会迅速退化成：

- 纯心情记录
- 与当前目录无关的随想
- 难以回看和再利用的碎片

所以最低约束只保留一条：

- 读者至少要能看出，这条反思是被什么客观现状触发的

不要求每条都证据闭环，但也不能完全脱离对象或主题现状。

## 6. 这层内容未来会流向哪里

`reflections.md` 不是终点，它是缓冲层。

其中的内容成熟后，通常会流向四处：

- 已压实结论 → 正文专题
- 已形成论证 → `evidence.md`
- 已形成稳定主线问题 → `deep-research-question-tree.md`
- 仍属缺口但值得追踪 → `backlog.md`

也就是说，反思层允许长期保留，但不意味着它永远不回流。

## 7. 一个典型误读信号

当 AI 开始说出下面这类话时，通常说明它已经偏离了这条规则的原意：

- “既然只是感悟，那随便放哪都行……”
- “既然有反思层，那正文里先写也没关系……”
- “既然不强调事实性，那就不需要现状锚点……”

这些说法的问题分别是：

- 让反思层重新失去稳定容器
- 让未压实判断提前污染正文
- 让反思层退化成与研究脱节的随想堆

## 8. 主规则应该保留什么，`intent/` 再补什么

主规则文件里，应该保留足够短、足够稳的内容，例如：

- 反思层属于研究辅助材料层
- 推荐使用 `reflections.md` / `notes/reflections.md`
- 允许记录基于现状的自由发散
- 不要把反思层直接当作正式正文或 Evidence 来源

这些信息应该足以让 AI 在多数场景下先执行对。

而 `intent/reflections.md` 补的是：

- 为什么研究需要这种低证据、但高启发性的缓冲层
- 为什么它不能被塞进 `source.md` 或 `evidence.md`
- 为什么它既自由，又仍要保留最小现状锚点
- 为什么它应留在 `notes/` 体系中，而不是额外升格

## 9. 本文件的使用方式

本文件不是默认入口。

只有在以下情况才读：

- 需要修改 `research-artifacts.md` 中关于 `reflections.md` 的条文
- 需要判断某条对象级/主题级感悟该放正文、Evidence、question tree 还是 reflections
- 发现 AI 反复把阶段性感悟写进正式正文
- 需要解释为什么这类内容可以更自由，但不能完全脱离现状

平时执行研究组织任务，仍优先读 `research-artifacts.md` 本身。
